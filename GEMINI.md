# GEMINI.md - Project Context & Directives

## Project Identity
**Name**: 3D Spring Emulator & Flexible Objects
**Goal**: Create a satisfying, realistic simulation of various flexible objects, starting with a doorstop spring.

## Key Characteristics
1.  **The "Boing"**: The physics must feel "springy" and satisfying.
2.  **Variety**: Support multiple types of flexible objects (Spring, Rope, Chain, etc.).
3.  **Interaction**: Mouse/Touch interaction to pull, drag, and flick objects.
4.  **Visuals**: High-quality materials and lighting.

## Technical Directives
- **Framework**: Vanilla Three.js.
- **Physics**: Cannon-es.
- **Architecture**: Modular design with a common interface for all flexible objects.
- **Code Style**: Modular, typed (TypeScript), clean.

## Object Types
1.  **Doorstop Spring**: Stiff, coiled, "boingy".
2.  **Rope**: Loose, flexible, non-elastic (mostly).
3.  **Chain**: Heavy, linked segments.
4.  **Soft Ball**: Squishy sphere (optional).
